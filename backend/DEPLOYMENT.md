# Render Deployment Guide

This guide explains how to deploy the Agent1o1 Laravel application to Render's free tier using Docker.

## Prerequisites

- A Render account (free tier)
- Git repository with the code
- PostgreSQL database (included in Render free tier)
- Docker installed locally (for testing)

## Deployment Steps

### 1. Push Code to Git

Ensure your code is pushed to a Git repository (GitHub, GitLab, or Bitbucket).

```bash
git add .
git commit -m "Add Render deployment configuration"
git push
```

### 2. Create Render Account

1. Go to [render.com](https://render.com)
2. Sign up for a free account
3. Connect your Git repository

### 3. Deploy Web Service

1. Click "New +" in Render dashboard
2. Select "Web Service"
3. Connect your repository
4. Render will automatically detect the `render.yaml` file
5. Review the configuration:
   - **Name**: agent1o1
   - **Environment**: Docker
   - **Plan**: Free
   - **Dockerfile**: ./Dockerfile

6. Click "Deploy Web Service"

### 4. Set Environment Variables

After the web service is created, add the following environment variables in the Render dashboard:

**Database Variables** (from your PostgreSQL database):
- `DB_CONNECTION=pgsql`
- `DB_HOST` (from Render database dashboard)
- `DB_PORT=5432`
- `DB_DATABASE` (from Render database dashboard)
- `DB_USERNAME` (from Render database dashboard)
- `DB_PASSWORD` (from Render database dashboard)

**Application Variables**:
- `APP_ENV=production`
- `APP_DEBUG=false`
- `APP_URL=https://agent1o1.onrender.com`
- `APP_KEY` (generate with `php artisan key:generate` locally)
- `LOG_CHANNEL=stack`
- `LOG_LEVEL=debug`
- `CACHE_DRIVER=file`
- `SESSION_DRIVER=file`
- `QUEUE_CONNECTION=database`
- `SESSION_LIFETIME=120`
- `BROADCAST_DRIVER=log`
- `FILESYSTEM_DISK=local`

**Passport OAuth Variables** (required for API authentication):
- `PASSPORT_PRIVATE_KEY` (generated by `php artisan passport:install`)
- `PASSPORT_PUBLIC_KEY` (generated by `php artisan passport:install`)
- `PASSPORT_PASSWORD_CLIENT_ID` (from oauth_clients table after passport:install)
- `PASSPORT_PASSWORD_CLIENT_SECRET` (from oauth_clients table after passport:install)
- `PASSPORT_CONNECTION` (leave empty to use default database connection)

### 5. Deploy Database

1. Click "New +" in Render dashboard
2. Select "PostgreSQL"
3. Name: `agent1o1-db`
4. Plan: Free
5. Click "Create Database"

### 6. Run Migrations

Migrations are included in the Dockerfile and run during the build process. However, the database must be deployed first before migrations can succeed.

**Important:** Deploy the PostgreSQL database first (step 5), then deploy the web service (step 3). The web service build will fail if the database doesn't exist.

If you need to run migrations manually after deployment:

**Via Render Shell:**
1. Go to your web service in Render dashboard
2. Click "Shell" tab
3. Run: `php artisan migrate --force`

### 7. Configure Passport

Generate Passport keys locally before deployment:

**Step 1: Generate keys locally**
```bash
php artisan passport:install
```

**Step 2: Extract the values**
```bash
cat storage/oauth-private.key
cat storage/oauth-public.key
php artisan tinker --execute="echo Laravel\Passport\Client::where('password_client', true)->first()->id;"
php artisan tinker --execute="echo Laravel\Passport\Client::where('password_client', true)->first()->secret;"
```

**Step 3: Add to Render environment variables**
- `PASSPORT_PRIVATE_KEY` - content of storage/oauth-private.key
- `PASSPORT_PUBLIC_KEY` - content of storage/oauth-public.key
- `PASSPORT_PASSWORD_CLIENT_ID` - client ID from above
- `PASSPORT_PASSWORD_CLIENT_SECRET` - client secret from above

**Important:** Never commit the actual keys to git. They should only be stored as environment variables in Render.

### 8. Update Database URL

Update the database connection in your environment variables to use the Render PostgreSQL connection string format:

```
DATABASE_URL=postgres://user:password@host:port/database
```

## Free Tier Limitations

Render's free tier has the following limitations:
- **Web Service**: Spins down after 15 minutes of inactivity (takes ~30 seconds to wake up)
- **Database**: 90 days of inactivity limit
- **Disk**: 1GB persistent storage
- **RAM**: 512MB
- **CPU**: Shared CPU

## Troubleshooting

### Application fails to start
- Check the Render logs for errors
- Ensure all environment variables are set correctly
- Verify database connection details

### Database connection errors
- Ensure the database is deployed and accessible
- Check that the database host, port, username, and password are correct
- Verify the database is not sleeping (wake it up by accessing it)

### Build failures
- Check the build logs in Render
- Ensure all dependencies are properly defined in `composer.json` and `package.json`
- Verify the build commands are correct

## Monitoring

- View logs in the Render dashboard under the "Logs" tab
- Monitor resource usage in the "Metrics" tab
- Set up alerts for errors or high resource usage

## Updates

To deploy updates:
1. Push changes to your Git repository
2. Render will automatically detect and redeploy
3. Monitor the deployment logs for any issues
